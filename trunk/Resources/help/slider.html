Slider Panel


A Cecilia module may have many sliders. They are stacked in the lower part of the interface. 




Value


The slider value can easily be consulted in the rightmost part of the slider:



The slider value also serves as an entry field, aloowing the user to type in the value of the slider.



Type


There are two types of sliders. The coloured slider and the grayed out slider.


The grayed out slider:



Sends a one time value at the beginning of the performance. Once the performance is started, it has no more effect on the sound. In order to make apply a change on it's value, you will have to stop the performance and start it again. The grayed out slider has no automation capabilities. Its value can be changed using the knob of the slider or by typing in the value in the entry field. The grayed out slider is particularly useful to define a value that won't change during the performance. For example, the number of band in a vocoder or a random seed in a stochastic module.


The coloured slider allows the users to modify a parameter's value during the performance. It constantly communicates its value so that parameter changes reflect the user movements. It aloso has automation capabilities. The coloured slider particularities are described here.



Control


Sliders (of any type) can easily be controlled with the mouse. 



A) Clicking and dragging the knob allows the user to modify the slider value in accordance with its gesture.


B) Clicking on the range of the slider will instantly move the knob to the mouse position. 


C) The value of a slider can be entered numerically. Click once on the slider value, it turns into an entry field. Write down the new value you want the slider to take. Note, the value won't be updated until you hit Enter or click outside of the entry field. The value can also be changed by scrolling the number while holding down the mouse button



Linking with the Graph


For each coloured slider, a graph line is created. This graph line represents the movement of the slider over the time of performance. It is a visual representation of a slider's evolution (the automation).


This graph line can be modified in the same way a regular graph line is. (see Graph section).


To edit the graph line associated to a slider, you must first select it. To quickly select it, click on the slider label:



Shift+click on the same label will isolate the line for the user to edit it easily.



Automation


A coloured slider's motion can be recorded, played back and modified. Automation mode is defined by those two symbols:



Recording

The motion of a slider can be recorded during a performance.To do so, the user clicks on the red button (mini rec button):



This particular slider will then be armed and ready to record the motion. Click the Play button in ythe Main Control Panel to start the performance. Move the slider. When the performance is done, the newly recorded automation will show up as a line segment in the Graph.


The resolution of the recorded automation can be ajusted using the mini-slider in the upper area of the Graph. 

For the same automation, it can either be very precise:



Or made approximate using data reduction:



Playback

Automation playback is activated with the mini play button. Clicking on it once will turn it green:



When set in this mode, the automation will be played back as the graph line displays it. The slider knob will follow this automation and move by itself. Click a second time on the mini play button and it will turn yellow. in that mode, the automation will be read but the slider know won't move, saving CPU usage.



Touch Record

The user can also activate both the recording and playback modes of the automation. This will create two different types of touch recording automations.
