cgen

a text-field for function table generation.


DESCRIPTION


cgen widget is a text-field allowing the user to enumerate a serie of values that will be used to generate a Csound table. The table will be created with the Csound GEN routine defined after the flag -gen. Note that a negative GEN number implies that the function is not rescaled between 0 and 1, and maintains its original values. The size of the table is set using the -size flag (usually a power of 2). The function table number is accessible in the orchestra with a variable named givariable_name. The -init flag sets the initial numerical value of the popup menu. Popup menu colour can be set using -col flag and a colour argument. The cgen appears in the lower left corner of the interface in the order listed. 


INTERFACE SYNTAX


cgen

(compulsory)

variable_name (mandatory, no spaces allowed)

-label Text showing in Cecilia slider pane (spaces allowed). Default value = ' '

(optional)

-init Initial value (numerical list). Default value = 1

-gen Csound generator used to create the table. Default = -2

-size Size of the table (in samples). Default = 8192

-col Colour of the slider (see Colour arguments). A default colour is applied by Cecilia

-help Text for tooltips (space allowed) Default value = ' '


ORCHESTRA SYNTAX

asig oscil kamp, kfreq, giWaveform

 

where giWaveform is a table containing a harmonic waveform created with the GEN10 routine

(In this case, the user has to define the proportion of each harmonic in the text-field of a cgen)


EXAMPLES


cgen custom -label Custom wave -gen 10 -size 8192 -init 1,0,0,.3,0,0,.2,0,.1,0,0,0,.05 -col red
