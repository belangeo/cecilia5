Output


With the Output panel, the user can control the attributes of the sound outputted by Cecilia. 



Output file


When pushing the Rec button in the Main Control panel, the output sound is written into a soundfile. This soundfile must be defined in the Filename field of the Output panel:



To do so, click on the folder icon of the Output Tools:



This action will open a Save as... dialog. Using that dialog, the user can define the path and filename of the soundfile that will be created on a rendering performance.


Note If more that one rendering performance is made using the same output filename, Cecilia will not overwrite the previous soundfile created. It will automatically create soundfiles with the same filename followed by the extension _00x, incrementing everytime a sound is created.


Output File Tools:

The Output panel includes a series of four tools:



With those tools the user can listen, edit, modify or reuse the output soundfile.


Clicking on the speaker icon will open the soundfile in the player application. It allows the user to hear the soundfile without any processing. The player application must be set in the Preference Panel.


The scissors icon opens the soundfile in an edition application. The soundfile can then be edited after being created in Cecilia. The edition application must be set in the Preference Panel.


The folder icon helps the user define the soundfile name and path that will be created by Cecilia. It opens  a save file dialog.


The recycle icon is a shortcut for the user to rapidly use the output soundfile as an input soundfile. Clicking on it will automatically replace the soundfile in the input panel by the one defined in the Output filename field.


Performance duration

 

The Cecilia performance is a timed process. Which basically means that the input sound will be processed for a fixed amount of time. This performance time can be set using the Duration slider in the Output panel:



As the user moves the Duration slider, the X-axis of the graph reajusts to the new performance duration. Shift-clicking on the knob of the slider lets the user type in a specific duration. The slider value can also reajust to the input sound duration when the hitting the clock icon (in the sampler window).


Output Gain


A final gain control is available to the user in the Output panel. The sound produced by Cecilia can sometimes be a too loud or not loud enough. The Gain slider gives the user the possibility to adjust the level of the signal right before it is sent to the DAC or the soundfile.



It has a range of 66dB, going from -48dB to +18dB. Shift-clicking on the knob of the slider lets the user type in a specific gain value. The slider value can also reajust to the peak value (refers to the Peak Value section)


VU Meter


A continuous VU Meter lets the user see the current level of the outcoming signal:



The VU Meter is a graphical interface presenting the level of a signal. Every horizontal bar in the VU Meter represents the signal level of one channel. From Cecilia's VU Meter, the user can read the value of a signal from the transition between the lighter and darker tone in an horizontal bar. Every division in that horizontal bar represents a 3dB step. 


The VU Meter automatically reajusts in size to the Output format.


Output Format


Cecilia can output sound in various formats. The Output Format can be chosen using the context menu in the Output panel:



Available standard formats are Mono, Stereo, Quad, 5.1 and Octo. The user can also customize the Output Format by clicking Custom... :



This output process is independant to the format of the input sound (thanks to Cecilia's sampler).


In the present version of Cecilia, the only two available formats are Mono and Stereo. Multichannel formats will soon be available. In the meantime, choosing a custom format, the user will have to build his own  Cecilia module that supports the specific number of channels.


Peak Value


The maximum value of the VU Meter is stored in a display called Peak:



That value is updated everytime the level of a outcoming signal go higher to the current Peak Value:



Optimal value is 0dB. The gain ajustment based on the Peak value can be automatically done by Cmd+click on the Peak Value. Double-clicking on the Peak Value will reset it to it's minimum value (-90dB).
