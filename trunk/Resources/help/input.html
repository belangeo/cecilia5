Loading a soundfile


To load a soundfile, click on the Folder icon in the tools. A file dialog asking to choose a soundfile will popup. After selecting it and clicking open, the filename will be added to the Input Panel:



When a sound is loaded, Cecilia automatically scans the whole folder where the file resides. Cecilia identifies all soundfiles in that directory and adds them to a contextual menu. This popup menu gives the user a quick access to all the soundfiles of a directory. The user only has to click on one of the soundfiles in the menu in order to rapidly load it as a source.



Tools


Next to the filename of the loaded soundfile, there is a group of 4 tools:



1) Clicking on the speaker icon will open the soundfile in the player application. It allows the user to hear the soundfile without any processing. The player application must be set in the Preference Panel.


2) The scissors icon opens the soundfile in an edition application. The soundfile can then be edited before being processed in Cecilia. The edition application must be set in the Preference Panel.


3) The folder icon allows the user to load a soundfile through a file dialog.


4) The arrow icon opens a window that lets the user set parameters of the sound. This window is dependant on the type of input defined by the module creator. It can either be a sampler window or a file input window.



Sampler


The sampler gives a great deal of control over the input soundfile. It is intended to mainly control the playback of the soundfile. It gives access to looping, gain control and transposition of the sound prior to entering Cecilia processes.


The sampler offers the possibility to load a soundfile whose number of channels  is different then the Cecilia module output. For example, the user could load a quadraphonic soundfile and, with the automatic assistance of the sampler, use it in a stereo module. The sampler uses a distribution algorithm to convert a non-matching format (say, quadraphonic) to the desired output format (stereo).


The sampler parameters can be recorded and played back with the automation system. This means that all parameters are attached to a graph line that can be edited.


If the input module used is a sampler (in most cases), the window opening by clicking the arrow tool will look like this:



In this window, a header describes the caracteristics of the soundfile loaded. It gives its absolute path, duration, file type, bit depth, number of channels and sample rate. 


A first slider (named Offset) allows the user to modify the starting point of reading in the soundfile.


The sampler normally plays the input soundfile in loop. The looping mode can be set using the context menu. Available looping modes are: 

- Off: The sound is played once and then stops.

- Forward: The sound is played repeatedly from the beginning to the end.

- Backward: The sound is played repeatedly from the end to the beginning.

- Back & Forth: The sound is played forward and then backward alternativelty.


The Start from loop toggle allows the user to set the starting point of reading in accordance with the Loop in slider.


A set of three tools in the sampler window is similar to the input tools described earlier. The speaker and scissors icons have the same functionality. The clock icon links the performance duration to the duration of the soundfile. Clicking on it will automatically reajust the duration slider in the Output Panel.


Three sliders define the loop parameters of an input source. The Loop In slider sets the time in the soundfile where a loop starts. The Loop Time slider sets the duration of a loop. The Loop X slider is linked to the crossfade time between two loops.


The Gain slider controls the input level of a sound.


The Transpo slider controls the direct transposition of the input soundfile. 



File input


If the input module used is a file input (for instance, in cases where the soundfile is to be input into a table with a GEN 1 function), the window opening by clicking the arrow tool will look like this:



In this window, a header describes the caracteristics of the soundfile loaded. It gives its absolute path, duration, file type, bit depth, number of channels and sample rate. A single slider offers the user to modify the starting point of reading in the soundfile.


A set of three tools in the file input window are similar to the input tools described earlier. The speaker and scissors icons have the same functionality. The clock icon links the performance duration to the duration of the soundfile. Clicking on it will automatically reajust the duration slider in the Output Panel.
