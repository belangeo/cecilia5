Building a Cecilia Interface

graphical interface controls.


DESCRIPTION


Cecilia allows you to control Csound with a series of creative widgets. Widgets are GUI controllers defined in the interface panel of the Cecilia editor using a simple syntax.

In the interface panel, you can define a list of widgets you wish to add to the Cecilia interface. Widget definition starts with its type, followed by a variable name so that its value can be retrieved in the orchestra. The definition ends with a series of flags and values parameterizing the widget. Each widget has a different set of flags depending on its behaviour. Some flags are compulsory, others are optionals. One widget is defined per line but a widget definition can take more than one line if each intermediate line ends with a double backslash (\\).

A widget's value is transfered to the Csound orchestra using the widget's global variable. This means you can retrieve a widget value in your orchestra using a standard gkvariable_name or givariable_name syntax (slightly different for the crange).Â 

WIDGET TYPES

- cbutton: a trigger button.

- cfilein: an input file menu.

- cgen: a text-field for function table generation.

- cgraph: a graph line.

- cpoly: a polyphony manager.

- cpopup: a popup (contextual) menu.

- crange: a range slider.

- csampler: a multichannel sampler with cross-fade looping.

- cslider: a horizontal slider.

- ctoggle: a checkbox.


SCRIPTING

In Cecilia, it is also possible to do scripting in the score editor. Scripting is based on Python. For more informations: Python Scripting
