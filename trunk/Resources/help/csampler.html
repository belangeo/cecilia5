csampler

a multichannel sampler with cross-fade looping.


DESCRIPTION

This menu allows the user to choose a soundfile for processing in the orchestra. Many csamplers can be defined in an interface. They will appear under the input label in the left side panel of the main window in the order they have been defined. When you choose a sound using this interface, Cecilia will scan the whole folder for soundfiles. A submenu containing all soundfiles present in the folder will allow a quick access to them. Looping, pitch and amplitude parameters of the loaded soundfile can be controlled by a csampler window which opens when toggling the small drop down triangle just besides the name of the sound.


To use a soundfile (loaded with csampler) in the orchestra, use the built-in UDO sampler with variable_name (between square brackets [ ]) as the first parameter. The sampler UDO reads a soundfile in consideration of all the csampler parameters. A sampler can have as many outputs as you wish aregardless of the number of channels in the soundfile. A distribution algorithm is used to assign x number of channels to y number of outputs.

For further informations on the use of csampler, refer to the Input section.


INTERFACE SYNTAX

csampler

(compulsory)

variable_name (mandatory, no spaces allowed)

(optional)

-label Text showing in Cecilia slider pane (spaces allowed). Default value = ''


ORCHESTRA SYNTAX

a1, a2, a3,... sampler  [variable_name], xamp, xpitch

 

xamp and xpitch are scaled from 0 to x, 1 being the normalized value. They are optional and default to 1.


EXAMPLES

In the interface panel :

csampler snd -label The sound input


In the orchestra :

asig1,asig2 sampler [snd], 1,1.25

asig1 sampler [snd], kamp,ipit

asig1,asig2,asig3,asig4 sampler [snd], .5,apit
