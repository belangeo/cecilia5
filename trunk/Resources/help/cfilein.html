cfilein

an input menu.


DESCRIPTION


This interactive menu allows the user to import a soundfile into the orchestra. When a file is chosen using this widget, Cecilia will scan the whole folder and create a dropdown menu with all the soundfiles within this folder to give a quick access to them.


Many cfileins are defined in an interface and appear under the input label in the left side panel of the main window in the order defined. 


In the orchestra, you use the variable_name (between square brackets [ ]) as a parameter of an opcode that requires a file path. The built-in UDO tables allows you to load a stereo soundfile into two Csound tables. The output of the UDO tables is the numbers of the two function tables.

For further informations on the use of cfilein, refer to the Input section.


INTERFACE SYNTAX


cfilein

(compulsory)

variable_name (mandatory, no spaces allowed)

(optional)

-label Text showing in Cecilia slider pane (spaces allowed). Default value = ''


ORCHESTRA SYNTAX


gi1, gi2 tables [variable_name], 

 

gi1 and gi2 are the numbers of the function tables containing the left and right channels of a stereo soundfile.


EXAMPLES


In the interface panel :

cfilein snd -label The sound input


In the orchestra :

gi1,gi2 tables [snd]

a1, a2 diskin2  [variable_name], kpitch

giTable ftgen 100, 0, 1500, 1, [variable_name]
