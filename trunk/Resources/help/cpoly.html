cpoly

a voice manager. 


DESCRIPTION


cpoly is a widget conceived to help manage the voice polyphony of a module. cpoly comes with a popup menu that allows the user to choose how many instances (voices) of a process will be simultaneously played. It also provides a mini slider to adjust the voice spread of those different voices. 


cpoly has two values that are passed to the orchestra: the number of voices and the voice spread. The number of voices can be collected using gkvariable_namenum. To access the voice spread use gkvariable_namespread.


This widget is aimed at advanced Cecilia interface builders. The use of cpoly requires that you build an orchestra taking advantage of its features. No documentation for now. Please refer to module ArcTanDisto.cec to see cpoly used in the way it is entended to. 


Note: the cpoly interface object and its associated variables can be used in the orchestra any way one sees fit. 


INTERFACE SYNTAX


cpoly

(compulsory)

variable_name (mandatory, no spaces allowed)

-label Text showing in Cecilia slider pane (spaces allowed). Default value = ''

(optional)

-min Minimum number of simultaneous voices. Default = 1

-max Maximum number of simultaneous voices. Default = 10

-init Initial number of simultaneous voices. Default value = min value

-help Text for tooltips (space allowed) Default value = ''
