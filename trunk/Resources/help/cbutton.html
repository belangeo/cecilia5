cbutton

a trigger button.


DESCRIPTION


The button appears in the lower left corner of the interface in the order listed. The value of the button is passed to the orchestra with a variable named gkvariable_name at k-rate. The value of the variable stays 0 until the button is pressed. When the button is pressed, the value changes to 1 for one pass if -trig is set to 0 or until the button is released if -trig is set to 1. Buttons only work at k rate. Button colour can be set using -col flag and a colour argument. 


INTERFACE SYNTAX


cbutton

(compulsory)

variable_name (mandatory, no spaces allowed)

-label Text showing in Cecilia slider pane (spaces allowed). Default value = ' '

(optional)

-trig Trigger mode. 0 = triggers for one k-pass, 1 = triggers until button released

-col Colour of the slider (see Colour arguments). A default colour is applied by Cecilia

-help Text for tooltips (space allowed) Default value = ' '


ORCHESTRA SYNTAX


if gktrigger==1 then

...

endif

 

where gktrigger is a trigger generating a Csound event.


EXAMPLES


cbutton noteTrigger -label Note Trigger  -init 1 -col olivegreen -help Generate a new note.
