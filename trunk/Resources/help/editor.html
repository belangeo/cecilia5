The Cecilia Editor


Cecilia is built around a simple and efficient Csound text editor. 


The editor opens autmatically when Cecilia is launched. Once you have chosen a module with the File/Modules menu, you can see its code by doing Cmd+E or going to the Window menu and selecting the Flip Editor/Interface option.



The editor is divided in three sections:

- Interface

- Orchestra

- Score


Each of these sections can be adjusted in size using the dividers separating them. They can also be minimized using the arrow next to their header.


To understand how to use of the Editor, you should be familiar with Csound programming. The online Csound Manual (http://www.csounds.com/manual/html/) is a good place to start.


Note: standard Csound (.csd) files can be loaded here. Go to File menu -> Open. In the load file dialog, enable the .csd files.


Interface


To build an interface, use the syntax explained here: Building an Interface. It allows the easy integration of various widgets to the Cecilia Interface.


Right-clicking in the Interface section shows a contextual menu with the various widgets availlable. This menu is a way of easily inserting the standard widgets to your interface.


Cecilia keywords will automatically be highlighted in yellow to let you know that you are using the right syntax. 


Orchestra


This section is where the Csound orchestra is written. Standard Csound syntax is used. However the Cecilia Editor offers a number of special features to facilitate the writing of well structured code. 


First, Cecilia identifies all Csound keywords in the text. It colorizes them depending on their function. Opcodes are colorized in blue. Instruments and Opcodes statements are colorized in red. Strings are colorized in green. Comments are colorized in olive.


To obtain information on a particular opcode, select the opcode in the text and hit Cmd+U. Or go to Csound menu and click Help with the selected opcode.  The Csound manual page for this opcode will load in your web browser (as defined in the Cecilia Preference Panel).


While writing Csound code, Auto-completion can be used if it is set the Format menu. Just type in the first few letters of an opcode and hit Cmd+K. A contextual menu appears and give you all possible opcodes starting with those letters. Browse through them with the arrows and select the right one by hitting the Return key.


When an opcode is identified in a line, the status bar (bottom part of the Editor) shows the standard syntax for that opcode. 


Right-clicking in the Orchestra section shows a contextual menu with all possible Csound opcodes.  This menu is a way of easily inserting an opcode with its standard syntax. 


Lines can conveniently be commented/uncommented with the Un/Comment line(s) function (in theFormat menu). Place your cursor on the line(s) you want commented and hit Cmd+J to invert their current state.


Csound code can be indented by going to the Format menu and selecting Indent Orchestra.


Finally, a file path can be added to an orchestra with the Insert file path... option (Cmd+L) in the Format menu. A dialog window appears to help you choose a file. The complete file path is automatically added (inside double quotes) at your cursor position. 



Score


The score section aloows the use of standard Csound scores and Python Scripted scores developped for Cecilia.


Cecilia identifies all Csound score statements in the text and colorizes them. Score statements are colorized in cyan. Comments are colorized in olive.


The Tabulate score option in the Format menu helps you organize your score. By hitting Shift+Cmd+I, Cecilia will automatically calculate the width of each column for an even distribution. A comment line is added to easily identify each column to its p# parameter.
