Using Cecilia


Cecilia was developped to simplify and extend the creative possibilities of the Csound audio engine. Cecilia comes with many original builtin modules for sound effects and synthesis. It also lets you create your own GUI (grapher, sliders, toggles, popup menus) using a simple syntax.


Cecilia can be also be used as a dedicated editor for regular Csound work. Cecilia's Editor offers chockful of features to facilitate and accelerate development work, including opcode insertion, syntax highlighting and on-line Csound manual.

The composer will undoubtedly find the python scripting extensions very useful for score generation. While this feature will be greatly expanded in the future, some very useful python primitives are offered to facilitate extensive algorithmic note generation.


Here is a simple description of the available features in Cecilia. Please, experiment and play around with the software. Once you know where the functionalities are, it becomes a very intuitive compositional tool.



Quick startÂ 


Once installed, Cecilia can be found will all your other applications.



Cecilia will open with the empty Cecilia editor.




At this point, you can either Build your own Cecilia module or load a built-in module.



Load a module

Cecilia delivers with a complete set of modules ready to process or synthesize sound. The built-in modules were conceived with great care and produce amazing result with just a few clicks.


To load a built-in module, go to File -> Modules menu. Modules are organized in submenus depending on the type of processing. Look around, try the different modules. Experimenting with the various parameter ranges will soon allow you to reach the desired results.



Using a module


Once a module is loaded, the Cecilia interface appears. Each module has it's very own interface, but the general look is always pretty much the same.


In order to use Cecilia modules, follow those steps:

A) For sound processing modules, you first need to load an input sound in the Input panel


B) Set the performance parameters in the Output Panel


C) Start the performance in the Main Control Panel


D) Play around with the Sliders to adjust the process to your needs
N.B. You can easily assign a MIDI controller to every slider


E) Create automation curves in the Graph


F) Add some processings to the output sound using the Post-Processing Panel


G)Have more than one setting for each modules with the Preset functionality


H) Record the output sound of your process to a soundfile using the record function of the Main Control Panel
