cgraph

a graph line.


DESCRIPTION


A graph line represents the evolution of a variable during a Cecilia performance. A graph only operates at k-rate. The value of the graph is passed to the orchestra with a variable named gkvariable_name. Only the variable_name and the -min and -max flags are compulsory. Others are optional. An initial straight line can be set at the -init value. But the -func flag overwrites the -init value. It defines an initial break-point line shaped with time/value pairs (floats). Time values are multiplied by the total_time of the process and must total 1. When used, the -gen flag writes the graph line in a Csound table numbered with the argument. The graph can then be used for any purpose in the orcehstra by recalling its number.The -col flag defines the color of the graph line using a colour argument.

NOTE: the -func tag MUST be the last one given in the line!


INTERFACE SYNTAX


cgraph

(compulsory)

variable_name (mandatory, no spaces allowed)

-label Text showing in Cecilia slider pane (spaces allowed). Default value = ''

-min Minimum value (numerical). Default value = 0

-max Maximum value (numerical). Default value = 1

(optional)

-rel Slider scaling {lin, log}. Default = lin

-gen Number of the Csound table in which the graph line is written in.

-size Table size (in samples). Default = 8192

-col Colour argument. Default = random

-init Initial value of the graph line (creates a straight line). Default = minimum value

-func Initial graph line in break-points (serie of time/value point). 


ORCHESTRA SYNTAX


asig oscil gkenv,440,1


gkenv  would be a value directly obtained from a graph named env. 


EXAMPLES


cgraph env -label Envelope -unit x -rel lin -min 0 -max 1 -col red -func 0 0 .00001 1 .99 1 1 0 

cgraph indexenveloppe -label Index Env Shape -rel log  -min .0001 -max 1  -gen 27 -col green -func 0 .6 .1 .051 1 .0001
